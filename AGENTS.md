# 도트말씨 홈페이지 작업 규칙

이 저장소는 도트말씨의 공개 홈페이지 소스다. 작업실 안에서는 상위
`../COMPANY_MANUAL.md`, `../PERMISSIONS.md`, `../STAFF.md`를 상속하며 이 문서는
웹사이트에 필요한 규칙만 강화한다.

## 정본과 상태

- 홈페이지 현재 상태: `STATUS.md`
- 프로젝트 상태 원문: 작업실 루트 `TEAM_STATUS.md`와 각 프로젝트 status source
- 브랜드 정본: 상위 `brand/BRAND_GUIDE.md`
- 공개 로고 복사본과 해시: `public/brand/brand-assets.json`

프로젝트 상태 문구를 바꾸기 전에는 반드시 기준일과 정본을 다시 확인한다.
`완성`, `정식판`, `공식`, `검증 완료`는 대응 증거 없이 사용하지 않는다.

## 허용과 금지

- 홈페이지 소스, 공개 문서, 승인된 로고 복사본만 이 저장소에 둔다.
- 원본·패치 ROM, BIOS, 세이브, 세이브스테이트, 비밀키, 쿠키, 개인정보를 넣지 않는다.
- 미승인 패치 다운로드, 브라우저 패처, 외부 분석 스크립트를 추가하지 않는다.
- 호출명은 역할 프로필로 설명하며 실제 인간 직원이나 상주 계정처럼 표현하지 않는다.
- 원작 로고·캐릭터·플랫폼 표장을 도트말씨 브랜드와 결합하지 않는다.
- 로컬 빌드 성공을 공개 배포 승인으로 해석하지 않는다.

## 변경 게이트

1. 정확한 상태와 콘텐츠를 작성한다.
2. 제작자와 다른 검수자가 사실성·접근성·브랜드를 독립 검토한다.
3. `npm test`, `npm run lint`, `npm exec tsc -- --noEmit`을 모두 통과한다.
4. 공개 배포는 정확한 대상·revision·채널을 지정한 사용자 승인 뒤 별도 수행한다.

모든 사용자 인터페이스는 키보드로 조작 가능해야 하며 320px부터 가로 넘침 없이
읽혀야 한다. 상태는 색만으로 구분하지 않고 항상 텍스트를 병기한다.
